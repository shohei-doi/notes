はじめに

僕はUbuntuをメインのOSとして使っています。 その理由としては

  • Windowsはデータ分析に向かないし、無駄な機能が多いように思え、
  • Macは高い

からです。

以下ではUbuntuをインストールした後の環境構築についてまとめます。

Ubuntuのインストール

Ubuntuを使う場合、Windowsと共存させるのが便利かと思います。 その方法としては

  1. 仮想マシン
  2. Windows Subsystem for Linux (WSL)
  3. デュアルブート

があります。 いずれも、Windowsの入っているマシンでUbuntuを使う点で、マシンの選択肢の幅が広いです。

仮想マシンはWindows上でVMwareやVirtualBoxなどを使って構築し、その中にUbuntuをインストールします。 Windows上で動くのでめんどくさい設定はいりませんが、マシンスペックを最大限活用することはできません。

デュアルブートとは同じマシンの中にWindowsとUbuntuを別々にインストールすることで、マシンの起動時にどちらを使用するかを決めます。 マシンスペックをフル活用できる一方で、ブートローダーを始めパソコンのコアな部分をいじるので危険性を伴います。

いずれにせよ、好みのLinux OSをダウンロードします。 とりあえず初めはUbuntuを使うのがいいかと思います。

  • LTSとは2年ないし5年の長期保証があるバージョンです。
  • それ以外は9ヶ月の保証しかありません。

仮想マシンでインストールする場合はそれぞれのソフトの使い方を参照してください。 デュアルブートする場合はUNetbootinやRufusなどを使って、USBをインストールメディアを作成します。

NVIDIAドライバ

インストール時と初回起動時はブート選択画面でeを押して、quiet splashをnomodesetに書き換える。 初回起動後に


sudo ubuntu-drivers autoinstall

を実行する。

Zenbook S

Zenbook Sにおける問題を解決します（こちらを参考）。

解像度の問題

解像度に1920:1080を追加します。


echo 'xrandr --newmode "1920x1080_60.00"  173.00  1920 2048 2248 2576  1080 1083 1088 1120 -hsync +vsync' >> .profile
echo 'xrandr --addmode eDP-1 1920x1080_60.00' >> .profile

スリープの問題

スリープをdeepにします。 以下のファイルを開き、


sudo nano /etc/default/grub

GRUB_CMDLINE_LINUX_DEFAULTにmem_sleep_default=deepを追加します。

設定を反映させて、再起動させます。


sudo update-grub2

日本語入力

Input Methodでibusを選択し、Region & LanguageでMozcを追加して最上位に設定する。

Thunderbird

Thunderbirdがプリインストールされていない場合、以下のコードでインストールします（こちらを参考）。


sudo add-apt-repository ppa:mozillateam/ppa
sudo apt-get update
sudo apt install thunderbird

Gnome Tweaks

Ubuntuの場合、まず、以下のコマンドでGnome TweaksとGnome Shellのホストコネクターをインストールする。


sudo apt install gnome-tweaks chrome-gnome-shell

Gnome Tweaksを起動し、

  • DesktopのShow IconsをOFFにする。
  • Top BarでApplication MenuをOFFにして、Battery PercentageとDateをONにする。

Gnome Extensionsのインストール

  • User Themes
  • AlternateTab
  • Sound Input & Output Device Chooser
  • Removable Drive Menu
  • Caffeine
  • Clipboard Indicator

フォントのインストール

type catcherを使ってGoogle Fontsからフォントをダウンロードしてインストールします。 font-managerを使ってGoogle Fontsからフォントをダウンロードしてインストールします。


sudo apt install typecatcher

Dropbox

公式サイトからダウンロード&インストールをします。

Dropboxに保存しているファイル数が膨大な場合、同期できないので、以下のコマンドを実行します。


echo fs.inotify.max_user_watches=100000 | sudo tee -a /etc/sysctl.conf; sudo sysctl -p
dropbox stop
dropbox start

Slackを公式サイトからダウンロード&インストールします。

  • ソフトウェアセンターでインストールすると日本語入力ができません。

Git

Gitがデフォルトでインストールされていない場合はインストールをします。


sudo apt install git

続いて、以下の初期設定を行います。


git config --global user.name "shohei doi"

R/RStudio

1つのバージョンの場合

Rを以下のコマンドでインストールします（参考）。


echo -e "\n## For R package"  | sudo tee -a /etc/apt/sources.list
echo "deb https://cran.rstudio.com/bin/linux/ubuntu $(lsb_release -cs)-cran35/" | sudo tee -a /etc/apt/sources.list
sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys E298A3A825C0D65DFD57CBB651716619E084DAB9
gpg --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys E298A3A825C0D65DFD57CBB651716619E084DAB9
gpg -a --export E298A3A825C0D65DFD57CBB651716619E084DAB9 | sudo apt-key add -
sudo apt update
sudo apt install r-base

複数のバージョンの場合

RStudioが公開しているコンパイル済みのRをインストールします。


sudo apt update
sudo apt install gdebi-core curl
export R_VERSION=3.6.2
curl -O https://cdn.rstudio.com/r/ubuntu-1804/pkgs/r-${R_VERSION}_1_amd64.deb
sudo gdebi r-${R_VERSION}_1_amd64.deb
sudo ln -s /opt/R/${R_VERSION}/bin/R /usr/local/bin/R
sudo ln -s /opt/R/${R_VERSION}/bin/Rscript /usr/local/bin/Rscript

Microsoft R Open

Microsoft R OpenはMicrosoftが提供するRディストリビューションです。 メリットとしてはBLASに高速なMKLが使用されている点などが挙げられます。

まず、ここからOSに対応するキーを取得します。 Ubuntu 18.04系ならば次の通りです。


curl https://packages.microsoft.com/keys/microsoft.asc | sudo apt-key add -
 sudo apt-add-repository https://packages.microsoft.com/ubuntu/18.04/prod
 sudo apt update

そして、R Openのファイルをダウンロードし、解凍したディレクトリでインストールコードを実行します。


cd Downloads/microsoft-r-open
sudo ./install.sh

R Openでは再現可能性のためにパッケージのリポジトリ(MRAN)の日付が固定されています。 最新のパッケージをインストールしたい場合はリポジトリをCRANに変更します。 具体的には以下の内容を.Rprofileという名前でホームディレクトリに保存します。


local({
  r <- getOption("repos")
  r["CRAN"] <- "https://cran.ism.ac.jp/"
  options(repos = r)
})

パッケージのインストールの準備

dotR <- file.path(Sys.getenv("HOME"), ".R")
if (!file.exists(dotR)) dir.create(dotR)
M <- file.path(dotR, "Makevars")
if (!file.exists(M)) file.create(M)
cat("\nCXX14FLAGS=-O3 -march=native -mtune=native -fPIC",
    "CXX14=g++", # or clang++ but you may need a version postfix
    file = M, sep = "\n", append = TRUE)

Python

UbuntuではPython2とPython3がデフォルトで入っているはずです。 なぜか、Atom上でpython3を使いたくてもpython2系のモジュールをインストールしないとうまく行きません。 正直、pythonの環境構築に関しては未だに分からないことが多いので間違いがあれば教えてください。


sudo apt install python-pip
pip install numpy pandas scikit-learn matplotlib ipykernel

sudo apt install python3-pip
pip3 install numpy pandas scikit-learn matplotlib ipykernel

これをすればpython2のモジュールを入れなくても大丈夫？（参考）


pip3 install ipykernel --upgrade
python3 -m ipykernel install --user

Anaconda

Anacondaはデータ分析用に特化したPythonディストリビューションです。 Anacondaを使うことには賛否両論ありますが、メリットの一つはBLASとして高速なMKLをデフォルトで使用している点です。

上記サイトから適当なバージョンのAnacondaをダウンロードして、


bash Anaconda3-2019.10-Linux-x86_64.sh

を実行します。

最後に、


source ~/.bashrc

でパスを通します。

以下のコマンドが実行できればインストールされています。


conda -V

conda自体のアップデートとパッケージ全体のアップデートは次の通りです。


conda update -n base conda
conda update --all

Julia

Juliaは最近、成長が著しい科学計算用プログラミングで、計算速度の速さとコードの書きやすさで有名です。 ダウンロードして、適当なフォルダで解凍して、以下のコマンドでシンボリックリンクを貼ります（Julia ver 1.2.0をDownloadsフォルダで解凍した場合）。


sudo ln -s ~/Downloads/julia-1.2.0/bin/julia /usr/local/bin/julia

JuliaからPythonを呼び出すためにPyCallをインストールしますが、普段使っているPythonを呼び出すためには、以下のコマンドでPythonのパスを確認します。


which python

そして、Juliaで次のようにしてPythonへのパスを通します。


ENV["PYTHON"] = "path/to/your/python"

その上で、package modeにしてPyCallをインストールします。


add PyCall

Rの場合は何もせずにRCallをインストールします。


add RCall

MeCab

日本語の形態素解析のためにMeCabとNEologdを入れます（参考）。


sudo apt install mecab libmecab-dev mecab-ipadic-utf8
cd ~/Downloads
git clone https://github.com/neologd/mecab-ipadic-neologd.git
cd mecab-ipadic-neologd
sudo bin/install-mecab-ipadic-neologd

/etc/mecabrcを編集してdicdir = /usr/lib/x86_64-linux-gnu/mecab/dic/mecab-ipadic-neologdとします。


sudo atom /etc/mecabrc

Python3で動かすためのモジュールをインストールします。


sudo apt install swig
sudo pip3 install mecab-python3
  • 参考サイトのコードではうまくNEologdにパスが通らなかったので、taggerで-d /usr/lib/x86_64-linux-gnu/mecab/dic/mecab-ipadic-neologdと直接指定しました。

RMeCabをインストールします。


install.packages("RMeCab", repos = "http://rmecab.jp/R", type = "source") 
install.packages("RcppMeCab")
  • RMeCabはCRANにはないので注意。

Apache Spark

分散コンピューティング用のフレームワークであるApache Sparkをインストールします（こことここを参考にしました）。

まず、Javaをインストールします。


sudo apt install openjdk-8-jdk
java --version

続いて、適当なディレクトリに移動して、Apache Sparkのファイルをダウンロード&展開します（バージョンを2.4.4でHadoop付きとします）。


cd /usr/local
sudo curl -O https://www-us.apache.org/dist/spark/spark-2.4.4/spark-2.4.4４-bin-hadoop2.7.tgz
sudo tar xvf spark-2.4.4-bin-hadoop2.7.tgz

最後に、シンボリックリンク作成し、パスを通します。


sudo ln -s /usr/local/spark-2.4.4-bin-hadoop2.7 /usr/local/spark
sudo atom ~/.profile

atomでは末尾に以下を追加します。


export SPARK_HOME=/usr/local/spark
export PATH=$PATH:$SPARK_HOME/bin

パスの変更を反映させます。


source ~/.profile

spark-shellで起動できれば成功（のはず）です。

Rパッケージ

RでApache Sparkを使うにはSparkRかsparklyrを使います。


install.packages("SparkR")
install.packages("sparklyr")

Pythonパッケージ

PythonでApache Sparkを使うにはpysparkを使います。


sudo pip3 install pyspark

Atom

VScodeに移りつつありますが、以下の理由でAtomも入れておきます。

  1. VScodeはroot権限でファイルを開くのがめんどくさい。
  2. Juliaの統合開発環境として採用されている。

Atomを公式サイトからダウンロード&インストールします。

  • 設定でEditorのSoft Wrapを有効にします。

一般向けパッケージ

  • minimap：ミニマップ
  • highlight-column：列のハイライト
  • file-icons：色付きアイコン

LaTeX向けパッケージ

  • latex：タイプセット
    • デフォルトのエンジンをxelatexにします。
    • Clean Patternsで.pdfを.xdvに変更します。
  • language-latex：シンタックス・ハイライト
  • wordcount：ワードカウント
  • ginger：英文法チェッカー

Python向けパッケージ

  • Hydrogen：Atom上でjupyterの操作
  • hydrogen-python：variable explorer
  • autocomplete-python：オートコンプリート（Kiteが素晴らしい）

RStudioと同様にCtrl + Enterで実行かつ次の行への移動を行うためにキーバインディングを以下のように追加します。


'.platform-win32 atom-text-editor:not([mini]), .platform-linux atom-text-editor:not([mini])':
  'ctrl-enter': 'hydrogen:run-and-move-down'

ついでに、.pyファイルをAtomを紐づけします。

Julia向けパッケージ

  • Juno：統合開発環境

RStudioと同様にCtrl + Enterで実行かつ次の行への移動を行うためにキーバインディングを以下のように追加します。


'.platform-win32 atom-text-editor[data-grammar="source julia"], .platform-linux atom-text-editor[data-grammar="source julia"]':
  'ctrl-enter': 'julia-client:run-and-move'

ついでに、.jlファイルとAtomを紐づけします。

R向けパッケージ

RStudioで十分ですが、AtomでHydrogen経由でRを使うこともできます（こちら）を参照）。


install.packages(c('repr', 'IRdisplay', 'evaluate', 'crayon', 'pbdZMQ', 'devtools', 'uuid', 'digest'))
devtools::install_github('IRkernel/IRkernel')

ターミナル上でRを起動して、以下を実行します。


IRkernel::installspec()
  • rbox：シンタックス・ハイライト、オートコンプリートなど
